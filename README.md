# Operon Labs: Autonomous Company #1 (v2)

Build and iterate a practical product that helps indie hackers wire up a VPS for MVP launches.

---

## Product Direction

The v2 direction is focused on one clear outcome:

> Help solo founders and indie hackers go from a fresh VPS to a working MVP environment with less friction.

This repository is no longer centered on broad "autonomous company" exploration. It is now product-oriented around a VPS setup assistant for MVP teams of one.

---

## Problem

Indie hackers often lose time on infrastructure setup before they can ship:

- picking a VPS baseline
- hardening server access
- configuring runtime and deployment basics
- wiring databases, reverse proxy, SSL, and backups
- documenting and repeating setup reliably

The result is slow launches, inconsistent environments, and avoidable production risk.

---

## v2 MVP Scope

The v2 MVP should deliver a guided, repeatable flow for wiring up a VPS for early-stage products.

Core scope:

- **VPS bootstrap guidance** for common providers and Linux-based servers
- **Secure-by-default baseline** (SSH, firewall, non-root flow, basic hardening)
- **MVP deployment foundation** (runtime, app process manager, reverse proxy, HTTPS)
- **Operational essentials** (logs, simple monitoring hooks, backups, rollback notes)
- **Reproducible runbook output** so founders can re-run setup with confidence

Out of scope for MVP:

- complex multi-region infrastructure
- enterprise orchestration platforms
- advanced SRE workflows beyond core MVP reliability needs

---

## Target User

- Indie hackers
- Solo technical founders
- Small MVP teams that need to ship quickly without deep DevOps specialization

---

## Success Criteria

The product direction is considered on track when:

- a new VPS can be prepared through the product workflow end-to-end
- setup time to first deploy is meaningfully reduced
- the produced setup is safer and more consistent than ad hoc manual steps
- users can follow generated guidance/runbooks without external consulting

---

## Repository Intent

This repo captures the evolving product specification, implementation, and iteration cycle for the v2 VPS-for-MVP direction.

High-level principles:

- prioritize practical shipping velocity for founders
- keep recommendations simple, explicit, and reproducible
- favor secure defaults over convenience shortcuts
- maintain clear documentation for each iteration
